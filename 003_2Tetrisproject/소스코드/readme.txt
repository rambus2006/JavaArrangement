DESCRIPTION :
-------------
MusicPlayer 는 자바 사운드 API의 클립과 외부 라이브러리인 
javazoom 의 Jlayer 를 사용한 음악 재생 프로그램입니다. 

현재는 wav 파일과 mp3 파일만 재생가능하고 wav 파일의 경우에는 
PCM_SIGNED 와 PCM_UNSIGNED 포맷의 16bit 비트레이트, 
그리고 48000hz 샘플링 레이트 이하의 음원만 재생이 가능합니다. 
이와 마찬가지로 mp3 도 MP3SPI.jar 를 사용하여 디코딩 과정을 거친후
PCM_SIGNED 포맷으로 클립에 불러들여 재생하고 있습니다. 
우리는 이러한 오디오 정보들을 AudioFormat 이라고 약속하고 있으며 
이러한 AudioFormat 에는 SamplingRate, BitRate, Channels(Stereo or Mono) 
등과 같은 정보가 있습니다.

이렇게 불러온 파일들은 파일리스트에 추가 되어 재생방식에 따라 순차재생,
반복재생, 임의재생 방식으로 리스트에 있는 목록을 가지고 연속 재생이 됩니다.

더불어 가사파일(음원과 같은 이름을 가진 txt 파일)이 존재할 경우에 가사창에
해당 내용이 출력되기도 합니다.

부가적인 기능으로 mp3의 경우 가수명과 제목을 가져오는 태그(id3.v2) 기능을 
javazoom 의 jlgui API를 내장하여 제공하고 있습니다.


FAQ : 
-----
- wav와 mp3 외의 확장자는 어떻게 재생하나요?
	자바 사운드 API 는 Service Provider Interface 를 제공하고 있는데
	이는 Sound API의 기능을 확장하는 인터페이스입니다. 
	mp3 는 원래 기본 확장자가 아니기 때문에 SPI를 통해 플러그인처럼
	사용해야합니다. 따라서 flac, ogg 같은 3th party 인코딩 음원은
	이를 제공하는 외부 라이브러리를 프로젝트에 내장하거나 직접 구현
	하셔야 합니다.

- wav 는 태그가 지원이 안 되나요?
	id3 태그는 mp3 를 위한 기능입니다. 자세한 건 mp3가 탄생한 배경을
	참고해보세요. 그러나 dbPower 와 같은 프로그램을 사용하면 wav 파일에도
	태그를 추가할 수 있습니다. 그러나 우리 프로그램은 해당 태그를 불러오지
	않을 것입니다.

- 왜 jmf 를 안 쓰고 javazoom 을 사용하였나요?
	jmf는 플러그인으로써 외부 라이브러리로 사용하는것이 불가능합니다.
	자바 개발 초기에는 코어파일로 사용되었으나 IBM과의 협력 개발에 있어
	불화가 생기는 바람에 개발도 중단되고 우여곡절을 많이 겪은 플러그인 입니다.
	그리고 워낙 미디어 쪽으로 특화 된 플러그인이라서 화상채팅이나 동영상 재생에
	더 자주 사용되는 것 같아 필요한 기능만 있는 javazoom 라이브러리를 사용했습니다.
	그런데 javazoom 이 조금 느리기 때문에 안드로이드 어플로 javazoom 을 사용하진 
	마시길 바랍니다.
